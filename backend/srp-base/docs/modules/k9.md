# K9 Module

The k9 module stores police dog assignments for characters.

## Feature flag

There is no feature flag for k9; the module is always enabled.

## Endpoints

| Method & Path | Description | Rate Limit | Auth | Idempotent | Request Body | Response |
|---|---|---|---|---|---|---|
| **GET `/v1/k9s/active`** | List all active K9 units. | 60/min per IP | Required | Yes | None | `{ ok, data: { k9s: K9Unit[] }, requestId, traceId }` |
| **GET `/v1/characters/{characterId}/k9s`** | List K9 units for a character. | 60/min per IP | Required | Yes | None | `{ ok, data: { k9s: K9Unit[] }, requestId, traceId }` |
| **POST `/v1/characters/{characterId}/k9s`** | Create a K9 unit. | 20/min per IP | Required | Yes (use `X-Idempotency-Key`) | `K9CreateRequest` | `{ ok, data: { k9: K9Unit }, requestId, traceId }` |
| **PATCH `/v1/characters/{characterId}/k9s/{k9Id}/active`** | Update active state. | 20/min per IP | Required | Yes (use `X-Idempotency-Key`) | `K9ActiveUpdateRequest` | `{ ok, data: { updated: number }, requestId, traceId }` |
| **DELETE `/v1/characters/{characterId}/k9s/{k9Id}`** | Retire a K9 unit. | 20/min per IP | Required | Yes (use `X-Idempotency-Key`) | None | `{ ok, data: { removed: number }, requestId, traceId }` |

### Schemas

* **K9Unit** –
  ```yaml
  id: integer
  characterId: integer
  name: string
  breed: string
  active: boolean
  createdAt: string (date-time)
  ```
* **K9CreateRequest** –
  ```yaml
  name: string
  breed: string
  ```
* **K9ActiveUpdateRequest** –
  ```yaml
  active: boolean
  ```

## Implementation details

* **Repository:** `src/repositories/k9Repository.js` persists K9 units.
* **Migration:** `src/migrations/057_add_k9_units.sql` creates the `k9_units` table.
* **Routes:** `src/routes/k9.routes.js` exposes the REST API and emits WebSocket/webhook events.
* **Tasks:** `src/tasks/k9.js` broadcasts active K9 units on a schedule.
* **OpenAPI:** `openapi/api.yaml` documents schemas and paths.
